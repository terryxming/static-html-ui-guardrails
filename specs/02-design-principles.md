# 设计原则

## 优先级

固定顺序：

1. 信息可信度
2. 阅读效率
3. UI 精致度

## Apple 风格转译

Apple 风格只能作为气质参考，必须转译为：

- 克制配色
- 精确排版
- 清晰层级
- 充足但不过度的留白
- 少装饰
- 强内容聚焦

不能转译为：

- 全屏 hero
- 产品营销页
- 卖点卡片
- 装饰性渐变
- 大图压过信息结构

## 高信息密度规则

- 允许牺牲一部分呼吸感来提升扫描效率。
- 必须保持层级清楚、段落可读、表格不拥挤。
- 密度由已有 HTML 的内容结构、信息量和组件复杂度决定，不由 AI 凭感觉决定。

## 默认服务对象

- 分析报告
- 数据仪表盘
- 知识长文
- 对比评估
- 项目交付说明
- 研究简报

这些类别只作为内容形态参考，用于辅助 UI 密度、组件呈现和交互适配判断；不得作为模板套用到用户内容上。

## 硬性禁止

- 禁止默认 hero 化。
- 禁止产品卖点式布局。
- 禁止无意义大图。
- 禁止装饰性渐变主导页面。
- 禁止一页全是浮动卡片。
- 禁止为了高级感强行添加复杂交互。
- 禁止为了统一视觉而改写、重排、删减、合并用户原始内容。
- 禁止让用户内容套入固定模板或固定页面骨架。
