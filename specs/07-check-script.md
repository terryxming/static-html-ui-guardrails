# 检查脚本规格

## 文件

```text
scripts/check_html_visual.js
```

## 输入

```text
node scripts/check_html_visual.js <html-file>
```

输入文件必须是用户已经产生的本地单 HTML 文件，脚本不得修改该文件。

## 输出

必须同时输出：

- JSON 报告
- Markdown 报告

所有用户可读的 `message`、`reason`、`suggestion` 和 Markdown 正文必须使用中文。

## 报告字段

每个检查项必须包含：

- `ruleId`
- `severity`：`blocker` / `warning` / `manual`
- `message`
- `location`
- `reason`
- `suggestion`

## 门禁分级

### blocker

失败后禁止交付。

包括：

- 不是单 HTML。
- 存在外部联网依赖。
- 缺少浅/深色 token。
- 缺少一键主题切换。
- 缺少内容结构保持声明或结构审计证据。
- 优化过程改写、重排、删减、合并用户原始内容。
- 存在把用户内容套入固定模板的结构性改造。
- 使用明显随机色值或随机字号。
- 存在营销 hero 化结构。
- 表格正文全 `nowrap`。
- 整页横向溢出。
- 关键交互不可用。
- Chrome DevTools MCP 验收发现控制台错误。
- 深色模式下关键文字不可读。

### warning

允许交付前修复，若不修复必须说明原因。

包括：

- 部分组件使用受控例外值但未注释。
- 某些图表色接近。
- 表格列宽需要人工优化。
- 页面密度判断可能偏保守。
- 结构保持审计只能人工确认，尚未提供程序化对照。
- `VQ-W001`：多个章节可能使用同一种白卡片样式。
- `VQ-W002`：目录链接较多但缺少导航布局强化。
- `VQ-W003`：表格数量较多但缺少紧凑数据表策略。
- `VQ-W004` / `VQ-W005`：标题字号比例不舒适。
- `VQ-W006`：未检测到足够的组件差异化。

### manual

需要人工复核。

包括：

- UI 是否足够克制。
- 页面是否仍有营销感。
- 内容形态参考是否符合真实用户任务。
- UI 优化是否有同化内容结构的风险。
- 图表是否适合该内容结构。
- `VQ-M001`：需要基于截图完成视觉质量评分。

## Token 检查

强制 token 化UI 属性：

- `color`
- `background`
- `border-color`
- `font-size`
- `line-height`
- `font-weight`
- `border-radius`
- `box-shadow`
- `gap`
- `padding`
- `margin`
- 图表色
- 状态色

允许受控例外：

- 媒体查询断点。
- 表格列宽。
- 图表条形百分比。
- SVG path 坐标。
- Canvas 绘制坐标。
- 与内容长度相关的 `minmax()` / `clamp()`。

## 外部依赖检查

必须检查：

- `<link href="http...">`
- `<script src="http...">`
- `<img src="http...">`
- `@import url(...)`
- `url(http...)`
- 外部字体请求
- CDN 域名

## 表格检查

必须检查：

- 是否存在 `.table-wrap` 或等价横向滚动容器。
- 是否存在表格正文全局 `white-space: nowrap`。
- 数字列是否右对齐。
- 长文本列是否允许换行。

## 主题检查

必须检查：

- 是否存在 light token。
- 是否存在 dark token。
- 是否存在主题切换按钮或等价控件。
- 是否存在 `data-theme` 或等价机制。

## 禁止项检查

必须检查：

- 大面积 hero。
- 装饰性渐变主导页面。
- 大面积随机阴影。
- 不受控圆角。
- 外链资源。

## 内容结构保持检查

必须要求执行者记录：

- 源文件路径。
- 输出副本路径。
- 是否覆盖源文件。
- 标题顺序是否保持。
- 正文文本是否保持。
- 表格数据是否保持。
- 章节层级是否保持。
- 引用、来源、口径说明是否保持。

默认规则：

- 不得覆盖源文件。
- 输出副本建议命名为 `<原文件名>.guarded.html`。
- 如果用户明确要求覆盖，必须在完成声明中写明用户授权。
