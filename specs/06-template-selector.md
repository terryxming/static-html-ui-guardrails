# 内容形态识别与 UI 适配参考

## 定位

本文件保留原 6 类页面经验，但降权为内容形态参考。

硬约束：

- 不存在旧式模板套用执行环节。
- 不得把用户内容套进模板。
- 不得改写、重排、删减、合并用户原始内容。
- UI 表现必须适应已有 HTML 的内容结构，而不是让内容适应 UI 模板。

## 五维识别模型

内容形态不是页面名称，而是五维观察结果：

1. 用户任务
2. 内容结构
3. 信息密度
4. UI 重心
5. 交互方式

## 判断顺序

1. 先读取已有 HTML，建立标题、段落、表格、图表、列表、引用、交互控件的结构清单。
2. 标记不可破坏边界：标题顺序、正文文本、表格数据、章节层级、引用内容、用户原始字段。
3. 根据内容结构和信息密度判断 UI 密度 token。
4. 根据表格、图表、目录、Tabs、筛选、折叠等已有或可安全增强的元素选择组件适配规则。
5. 交互只能作为最后增强，且不得隐藏关键结论、关键风险或核心数据。

## 关键规则

- 页面名称只是弱信号。
- 用户说“报告”不能直接套报告UI 结构。
- 如果用户任务不清楚，必须标记假设或追问。
- 内容形态参考不能反向决定内容结构。
- 交互不能为了高级感强行添加。
- 任何 UI 优化都必须能通过内容结构保持审计。

## 六类内容形态参考

### 分析报告

常见任务：

- 解释问题。
- 呈现证据。
- 输出结论和建议。

默认密度参考：`compact`

可适配 UI：

- 报告头
- 目录
- 章节
- 摘要卡
- KPI
- 图表
- 数据表
- 结论证据链
- 风险提示

禁止：

- 将用户原有章节改写成统一模板章节。
- 为了形成“报告模板”而移动证据、结论或风险位置。

### 数据仪表盘

常见任务：

- 状态监控。
- 指标扫描。
- 数据比较。

默认密度参考：`dense`

可适配 UI：

- KPI 网格
- 趋势图
- 异常状态
- 筛选栏
- 高密度表格

禁止：

- 为了仪表盘感删除解释文字。
- 隐藏原始表格口径或说明。

### 知识长文

常见任务：

- 沉淀概念。
- 方法。
- 教程。
- 复盘。

默认密度参考：`comfortable`

可适配 UI：

- 阅读型正文
- 目录
- 引用块
- 方法步骤
- 风险或注意事项

禁止：

- 强行加入 KPI 或复杂交互。
- 把连续阅读内容切碎成卡片堆叠。

### 对比评估

常见任务：

- 多对象比较。
- 多方案比较。
- 多维度选择。

默认密度参考：`compact`

可适配 UI：

- 对比矩阵
- 指标列
- 推荐标记
- 结论证据链

禁止：

- 为了对齐矩阵而删减原始维度。
- 把不同口径的数据强行放进同一比较结构。

### 项目交付说明

常见任务：

- 说明文件。
- 使用方式。
- 验收方法。
- 交付边界。

默认密度参考：`compact`

可适配 UI：

- 文件清单
- 步骤说明
- 验收清单
- 风险提示
- 状态徽标

禁止：

- 重排交付顺序导致验收路径变化。
- 合并原本分开的边界、风险和步骤。

### 研究简报

常见任务：

- 汇总问题。
- 关键发现。
- 证据。
- 下一步行动。

默认密度参考：`compact`

可适配 UI：

- 短摘要
- 关键发现
- 证据引用
- 下一步行动

禁止：

- 为了简短而删减关键证据。
- 将不确定信息改写成确定结论。

## 混合形态规则

如果页面同时符合多个内容形态：

- 保持原始内容结构。
- 按主要内容密度选择基础 UI token。
- 对局部模块使用局部组件适配。

示例：

- 分析报告中出现仪表盘数据概览模块：只对该模块使用高密度 KPI 或图表适配。
- 研究简报中出现对比矩阵：只对矩阵区域使用对比表格适配。
- 项目交付说明中出现验收清单：只对清单区域使用状态和勾选适配。
