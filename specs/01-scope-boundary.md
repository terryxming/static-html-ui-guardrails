# 范围与边界

## 唯一目标

本 Skill 只做一件事：

确保用户已经产生的本地静态 HTML 文件在不破坏内容结构的前提下，完成 UI 表现、组件呈现、交互状态和验收约束优化。

## 负责范围

- 接收用户已经产生的本地单 HTML 文件作为输入。
- 默认输出优化副本，例如 `<原文件名>.guarded.html`；除非用户明确要求覆盖，否则不得覆盖源文件。
- 固定本地单 HTML 页面 UI 表现，避免 AI 随机发挥。
- 固定样式 token、组件适配规则、内容形态参考和密度判断。
- 建立源文件内容结构清单，标记不可破坏边界。
- 确保单 HTML、离线、Windows Chrome 双击打开。
- 确保桌面端 UI 稳定，覆盖 `1280 / 1366 / 1440 / 1536 / 1920`。
- 确保表格、图表、目录、Tabs、筛选、折叠等组件视觉和交互稳定。
- 支持浅色/深色模式，并提供一键切换。
- 通过脚本检查和 Chrome DevTools MCP 浏览器验收拦住 UI 漂移、外部依赖、布局溢出、组件违规和内容结构破坏。
- 面向大量外部用户发布，文档必须自包含。

## 非范围

- 不验证数据真实性。
- 不判断业务结论是否正确。
- 不补充缺失事实。
- 不做数据分析逻辑。
- 不负责报告内容质量。
- 不改变、不破坏源文件的内容结构。
- 不改写、不重排、不删减、不合并用户原始内容。
- 不设置旧式模板套用执行环节。
- 不把用户内容套入固定模板或固定页面骨架。
- 不做移动端完整规范。
- 不做多文件站点。
- 不做需要服务器、构建工具或外部依赖的 Web 应用。

## 关键边界

- 内容来自用户已经产生的 HTML 文件，本 Skill 只约束内容如何被 UI 呈现。
- 本 Skill 不对内容真假负责。
- 本 Skill 可以在不改变原始内容结构的前提下，让 UI 层更清楚地呈现“来源、口径、说明”等信息，但不验证这些信息是否真实。
- UI 表现必须适应内容，不得让内容套用视觉模板。
- 若用户要求事实核查，应交给其他 Skill 或上游流程。
