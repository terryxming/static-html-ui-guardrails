# 组件与内容形态参考规格：本地静态 HTML UI 优化约束

本文件补充正式 PRD 中未展开的组件系统和内容形态参考。下游实现不得只列组件名称，必须为每个组件提供 UI 适配片段、适用场景、禁止用法和验收标准。

硬约束：

- 输入端是用户已经产生的本地单 HTML 文件。
- UI 优化必须原位适应已有内容结构。
- 不得改写、重排、删减、合并用户原始内容。
- 不存在旧式模板套用执行环节。
- 不得把用户内容套进固定模板或固定页面骨架。

## 1. 组件系统总规则

组件分为六类：

1. 布局组件
2. 阅读组件
3. 数据组件
4. 分析组件
5. 交互组件
6. 反馈组件

每个组件必须定义：

- UI 适配结构
- 适用场景
- 禁止用法
- 验收标准

组件视觉规则：

- 低装饰
- 强结构
- 少阴影
- 优先边框和背景分层
- 不为高级感强行加交互
- 不隐藏关键结论、关键风险、核心数据

## 2. 布局组件

### 2.1 Page Shell 页面框架

默认结构：

```html
<div class="app-shell">
  <aside class="sidebar">
    <nav class="toc"></nav>
  </aside>
  <main class="content">
    <div class="page"></div>
  </main>
</div>
```

适用：

- 分析报告
- 知识长文
- 研究简报
- 项目交付说明

例外：

- 仪表盘可使用顶部工具栏 + 宽内容区。
- 短页面可不显示目录。

禁止：

- 全屏 hero。
- 营销首页结构。
- 超长页面无导航。

验收：

- 1280px 下不横向炸版。
- 侧栏不遮挡主内容。
- 主内容宽度符合模板密度。

### 2.2 Report Header 页面头部

固定结构：

```html
<header class="report-header">
  <p class="eyebrow"></p>
  <h1></h1>
  <p class="lead"></p>
  <dl class="meta-list"></dl>
</header>
```

允许：

- eyebrow
- title
- lead summary
- metadata
- optional actions

禁止：

- 大图背景。
- 渐变主 UI。
- 产品卖点区。
- 夸张大标题。
- 首屏占满。

### 2.3 Section 章节

固定结构：

```html
<section class="section" id="">
  <div class="section-head">
    <h2></h2>
    <p class="section-note"></p>
  </div>
  <div class="section-body"></div>
</section>
```

规则：

- Section 是页面结构，不是 Card。
- 文档型页面禁止每个 section 默认套大卡片。
- 仪表盘允许 panel 化，但禁止装饰性卡片堆叠。

### 2.4 TOC 目录

触发条件：

- 超过 4 个一级章节。
- 或页面预期滚动超过 2 屏。

规则：

- 默认左侧粘性目录。
- 最多两级。
- 只放短标题。
- 不放解释性长文本。

## 3. 阅读组件

### 3.1 摘要卡 Summary Card

用途：

- 承载 1 个要点或 1 组短事实。

禁止：

- 长段落。
- 装饰插图。
- 过大字号。

### 3.2 结论块 Insight Block

用途：

- 承载判断和建议动作。

结构：

```html
<article class="insight">
  <strong></strong>
  <p></p>
</article>
```

禁止：

- 空泛口号。
- 营销卖点。
- 无证据的断言。

### 3.3 引用块 Quote

用途：

- 证据。
- 用户原话。
- 来源片段。

禁止：

- 装饰性金句。
- 与结论无关的引用。

### 3.4 风险提示 Risk Notice

用途：

- 风险。
- 限制。
- 注意事项。
- 缺失信息。

禁止：

- 替代普通段落。
- 替代主结论。
- 用高饱和颜色制造紧张感。

## 4. 数据组件

### 4.1 KPI Card

必须包含：

- 指标名
- 数值
- 单位或变化
- 口径说明

结构：

```html
<article class="card kpi">
  <span class="label"></span>
  <div class="value"></div>
  <p class="desc"></p>
</article>
```

禁止：

- 只有大数字没有口径。
- 营销式冲击字号。
- 一页堆太多 KPI。

验收：

- 数字等宽。
- 单位清晰。
- 无口径时标记“口径未提供”。

### 4.2 Data Table

用途：

- 明细。
- 证据。
- 对比项。

必须支持：

- 横向滚动。
- 长文本换行。
- 数字右对齐。
- 单位明确。
- 列宽受控。

禁止：

- 正文全部 `nowrap`。
- 长文本无提示截断。
- 无表头。
- 无单位。
- 列宽随机。

### 4.3 Chart Container

默认实现：

- CSS 图表。
- SVG 图表。
- 原生 Canvas。

禁止：

- 默认引入外部图表库。
- 使用无坐标、无说明的装饰图。
- 颜色超过图表色板。

首批支持：

- 柱状图。
- 条形图。
- 折线图。
- 环形图。
- 热力摘要。

复杂图表处理：

- 降级为表格 + 简化图。
- 或由本地内联实现。

### 4.4 Comparison Matrix

用途：

- 多对象对比。
- 多方案评估。
- 多维度选择。

规则：

- 作为独立组件，不是普通 grid/card 变体。
- 必须有明确对比维度。
- 每格内容必须短。

禁止：

- 每格长段落。
- 没有评价维度。
- 把矩阵当普通排版网格。

## 5. 分析组件

### 5.1 Insight + Evidence 结论证据链

关键结论必须绑定证据：

- 表格。
- 图表。
- 引用。
- 来源说明。
- 明确的用户输入。

注意：

- 本 Skill 不验证事实真假。
- 但UI 结构必须支持结论和证据形成清晰关系。

禁止：

- 结论先行但无证据结构。
- 把示例视觉做成真实业务结论。

## 6. 交互组件

### 6.1 Filter Bar

允许：

- 本地搜索。
- 轻量筛选。
- 分组选项。

禁止：

- 复杂查询构建器。
- 需要后端的筛选逻辑。
- 页面刷新依赖。

### 6.2 Tabs

用途：

- 同层级内容切换。

禁止：

- 隐藏关键结论。
- 隐藏关键风险。
- 隐藏核心数据。
- 把不同任务混在同一组 tabs。

### 6.3 Disclosure / Accordion

用途：

- 补充信息。
- 方法说明。
- 明细解释。

禁止：

- 默认折叠关键结论。
- 默认折叠关键风险。
- 默认折叠关键数据。

## 7. 反馈组件

### 7.1 Badge / Status

用途：

- P0 / P1
- 上升 / 下降
- 已确认 / 待确认
- 事实 / 推测 / 示例

规则：

- 文本必须短。
- 语义固定。
- 颜色映射固定。

禁止：

- 承载长句解释。
- 随机颜色。
- 状态语义不一致。

### 7.2 Alert / Notice

用途：

- 风险。
- 限制。
- 缺失信息。
- 假设。
- 错误状态。

禁止：

- 普通强调都做成 alert。
- 到处堆提示框。

## 8. 内容形态识别

内容形态不是页面名称，而是以下五维观察结果：

1. 用户任务
2. 内容结构
3. 信息密度
4. UI 重心
5. 交互方式

判断顺序：

1. 读取已有 HTML，建立标题、段落、表格、图表、列表、引用、交互控件的结构清单。
2. 标记不可破坏边界：标题顺序、正文文本、表格数据、章节层级、引用内容、用户原始字段。
3. 根据内容结构和信息密度判断 UI 密度 token。
4. 根据已有或可安全增强的元素选择组件适配规则。
5. 交互只能作为最后增强，且不得隐藏关键结论、关键风险或核心数据。

规则：

- 页面名称只是弱信号。
- 用户说“报告”不能直接套报告结构。
- 内容形态参考不能反向决定内容结构。
- 如果任务不清楚，必须标记假设或追问。

## 9. 首批 6 类内容形态参考

### 9.1 分析报告

常见任务：

- 解释问题。
- 呈现证据。
- 输出结论和建议。

默认密度参考：`compact`

可适配 UI：

- 报告头
- 目录
- 章节
- 摘要卡
- KPI
- 图表
- 数据表
- 结论证据链
- 风险提示

禁止：

- 将用户原有章节改写成统一模板章节。
- 为了形成“报告模板”而移动证据、结论或风险位置。

### 9.2 数据仪表盘

常见任务：

- 状态监控。
- 指标扫描。
- 数据比较。

默认密度参考：`dense`

可适配 UI：

- KPI 网格
- 趋势图
- 异常状态
- 筛选栏
- 高密度表格

禁止：

- 为了仪表盘感删除解释文字。
- 隐藏原始表格口径或说明。

### 9.3 知识长文

常见任务：

- 沉淀概念。
- 方法。
- 教程。
- 复盘。

默认密度参考：`comfortable`

可适配 UI：

- 阅读型正文
- 目录
- 引用块
- 方法步骤
- 风险或注意事项

禁止：

- 强行加入 KPI 或复杂交互。
- 把连续阅读内容切碎成卡片堆叠。

### 9.4 对比评估

常见任务：

- 多对象比较。
- 多方案比较。
- 多维度选择。

默认密度参考：`compact`

可适配 UI：

- 对比矩阵
- 指标列
- 推荐标记
- 结论证据链

禁止：

- 为了对齐矩阵而删减原始维度。
- 把不同口径的数据强行放进同一比较结构。

### 9.5 项目交付说明

常见任务：

- 说明文件。
- 使用方式。
- 验收方法。
- 交付边界。

默认密度参考：`compact`

可适配 UI：

- 文件清单。
- 步骤说明。
- 验收清单。
- 风险提示。
- 状态徽标。

禁止：

- 重排交付顺序导致验收路径变化。
- 合并原本分开的边界、风险和步骤。

### 9.6 研究简报

常见任务：

- 汇总问题。
- 关键发现。
- 证据。
- 下一步行动。

默认密度参考：`compact`

可适配 UI：

- 短摘要。
- 关键发现。
- 证据引用。
- 下一步行动。

禁止：

- 为了简短而删减关键证据。
- 将不确定信息改写成确定结论。

## 10. 混合形态规则

如果页面同时符合多个内容形态：

- 保持原始内容结构。
- 按主要内容密度选择基础 UI token。
- 对局部模块使用局部组件适配。

示例：

- 分析报告中出现仪表盘数据概览模块：只对该模块使用高密度 KPI 或图表适配。
- 研究简报中出现对比矩阵：只对矩阵区域使用对比表格适配。
