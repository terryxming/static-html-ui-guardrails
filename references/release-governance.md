# 发布与治理规格

## 外部发布定位

本 Skill 完成后会发布给大量外部用户使用。

因此正式交付必须：

- 自包含。
- 不依赖本轮对话上下文。
- 有清晰安装说明。
- 有快速开始。
- 有典型调用示例。
- 有示例输出。
- 有兼容性说明。
- 有升级说明。
- 有故障排查入口。
- 明确默认输入为用户已有本地单 HTML 文件。
- 明确默认输出优化副本，不覆盖源文件。
- 明确所有用户可见内容使用中文。

## 支持环境

核心支持：

- Windows
- Chrome
- 本地单 HTML

其他环境：

- macOS / Linux 如支持，必须标记为兼容或未验证。

## CHANGELOG

必须包含：

```text
CHANGELOG.md
```

每次修改以下内容都必须更新：

- Skill 行为。
- 执行流程。
- 设计原则。
- token。
- 组件。
- 模板。
- 检查脚本。
- 验收标准。
- 发布文档。
- 内容结构保持规则。
- 中文化输出规则。

记录字段：

- 版本。
- 日期。
- 变更内容。
- 变更原因。
- 兼容性影响。

## 使用后复盘

完整使用一次 Skill 后，必须强制询问：

```text
是否复盘本次 Skill 使用质量？请回复“是”或“否”。
```

用户选择“否”：

- 跳过复盘。
- 不生成复盘报告。
- 不修改 Skill。

用户选择“是”：

复盘报告必须包括：

- 本次 Skill 是否稳定执行。
- 做得好的地方。
- 做得不好的地方。
- 为什么做得不好。
- 可升级优化的地方。
- 对上游输入的影响。
- 对下游执行的影响。
- 建议改动文件。
- 预期结果。

## 完成声明

每次交付优化后的 HTML，完成声明必须使用中文，并包含：

- 源文件路径。
- 输出文件路径。
- 是否覆盖源文件；如覆盖，必须说明用户授权。
- 是否保持标题顺序、正文文本、表格数据、章节层级。
- 脚本检查结果。
- Chrome DevTools MCP 验收结果。
- 视觉质量评分总分和单项分。
- 截图证据路径：第一屏、表格区、长文区、深色模式。
- 是否发生二次优化；如发生，说明原因和修改内容。
- 未修复 warning/manual 项及原因。
- 是否已经询问用户复盘。

硬约束：

- 复盘只能先输出建议。
- 未经用户明确同意，禁止根据复盘自动迭代、修改或优化 Skill。
- 用户必须再次明确确认“进入 Skill 迭代”后，才允许修改 Skill。

## 破坏性变更

以下属于破坏性变更：

- 修改默认 token 值导致页面 UI 明显变化。
- 修改内容形态识别逻辑。
- 修改组件 HTML 结构。
- 修改检查脚本 blocker 规则。
- 修改浏览器验收要求。
- 删除已发布模板或组件。

破坏性变更必须：

- 在 `CHANGELOG.md` 标记。
- 说明迁移方式。
- 说明兼容性影响。
