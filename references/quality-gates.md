# 执行与质量门禁规格：本地静态 HTML UI 优化约束

本文件是辅助阅读文件，正式执行细节以 `specs/07`、`specs/08`、`specs/09`、`specs/10` 为准。

## 1. 唯一目标

`static-html-ui-guardrails` 只做一件事：

接收用户已经产生的本地单 HTML 文件，在不破坏源文件内容结构的前提下，对 UI 表现、组件呈现、交互状态和验收约束进行优化。

不做：

- 不验证数据真实性。
- 不判断业务结论是否正确。
- 不做数据分析逻辑。
- 不补充缺失事实。
- 不负责报告内容质量。
- 不改变、不破坏源文件内容结构。
- 不改写、不重排、不删减、不合并用户原始内容。
- 不设置旧式模板套用执行环节。
- 不做移动端完整规范。
- 不做多文件站点或服务器应用。

## 2. 固定优化流水线

每次优化已有 HTML 必须经过：

```text
读取已有 HTML
→ 建立内容结构清单
→ 标记不可破坏边界
→ 注入/整理 token 化 CSS
→ 原位优化 UI 与组件表现
→ 补充本地交互与主题切换
→ 脚本硬检查
→ Chrome DevTools MCP 浅色验收
→ Chrome DevTools MCP 深色验收
→ 结构保持审计
→ 修复并重跑
→ 输出证据与完成声明
```

禁止：

- 跳过内容结构清单。
- 跳过不可破坏边界标记。
- 跳过 token 应用。
- 跳过脚本检查。
- 跳过浏览器验收。
- 检查失败后用解释代替修复。
- 把用户内容套入固定模板或固定页面骨架。

## 3. 检查脚本

脚本文件：

```text
scripts/check_html_visual.js
```

输入：

```text
用户已经产生的本地 HTML 文件路径
```

默认输出：

```text
<原文件名>.guarded.html
JSON 报告
Markdown 报告
```

除非用户明确要求覆盖，否则不得覆盖源文件。

报告必须包含：

- 检查项编号。
- 严重级别：`blocker` / `warning` / `manual`。
- 文件位置或 DOM/CSS 线索。
- 失败原因。
- 修复建议。

所有用户可读内容必须使用中文。

## 4. 质量门禁分级

### 4.1 blocker

失败后禁止交付。

包括：

- 不是单 HTML。
- 存在外部联网依赖。
- 缺少浅/深色 token。
- 缺少一键主题切换。
- 缺少内容结构保持声明或结构审计证据。
- 优化过程改写、重排、删减、合并用户原始内容。
- 存在把用户内容套入固定模板的结构性改造。
- 使用明显随机色值或随机字号。
- 存在营销 hero 化结构。
- 表格正文全 `nowrap`。
- 整页横向溢出。
- 关键交互不可用。
- Chrome DevTools MCP 验收发现控制台错误。
- 深色模式下关键文字不可读。

### 4.2 warning

允许交付前修复，若不修复必须说明原因。

包括：

- 部分组件使用受控例外值但未注释。
- 某些图表色接近。
- 表格列宽需要人工优化。
- 页面密度判断可能偏保守。
- 结构保持审计只能人工确认，尚未提供程序化对照。

### 4.3 manual

需要人工复核。

包括：

- UI 是否足够克制。
- 页面是否仍有营销感。
- 内容形态参考是否符合真实用户任务。
- UI 优化是否有同化内容结构的风险。
- 图表是否适合该内容结构。

## 5. Chrome DevTools MCP 浏览器验收

必须检查：

- 页面能在 Chrome 打开。
- 源文件路径和输出副本路径已记录。
- 无外部网络请求。
- 无控制台错误。
- 无整页横向溢出。
- 浅色模式截图正常。
- 深色模式截图正常。
- 一键主题切换可用。
- Tabs 可用。
- 折叠面板可用。
- 表格搜索/筛选可用。
- 表格长文本不被无提示截断。
- 数字列右对齐且等宽。
- 优化副本与源 HTML 的标题顺序、正文文本、表格数据、章节层级保持一致。

桌面宽度：

- `1280px`
- `1366px`
- `1440px`
- `1536px`
- `1920px`

正式 Skill 至少包含：

```text
   ├─ quality-gates.md
   ├─ chrome-devtools-mcp-acceptance.md
   └─ release-governance.md
```

## 7. 完成声明

每次交付优化后的 HTML，完成声明必须使用中文，并包含：

- 源文件路径。
- 输出文件路径。
- 是否覆盖源文件；如覆盖，必须说明用户授权。
- 是否保持标题顺序、正文文本、表格数据、章节层级。
- 脚本检查结果。
- Chrome DevTools MCP 验收结果。
- 未修复 warning/manual 项及原因。
- 是否已经询问用户复盘。

## 8. 使用后复盘

完整使用一次 Skill 后，必须强制询问用户：

```text
是否复盘本次 Skill 使用质量？请回复“是”或“否”。
```

用户选择“否”：

- 跳过复盘。
- 不生成复盘报告。
- 不修改 Skill。

用户选择“是”：

复盘只能先输出建议。未经用户明确同意，禁止根据复盘自动迭代、修改或优化 Skill。

## 9. 交付前最终检查清单

交付 HTML 前必须确认：

- 是单 HTML。
- 默认未覆盖源文件，或已记录用户覆盖授权。
- 标题顺序、正文文本、表格数据、章节层级保持一致。
- 无联网依赖。
- 使用固定 token。
- 支持浅色/深色。
- 一键切换可用。
- 无营销 hero。
- 无整页横向溢出。
- 表格长文本可读。
- 关键交互可用。
- Chrome DevTools MCP 截图已生成。
- JSON + Markdown 检查报告已生成。
- blocker 为 0。
