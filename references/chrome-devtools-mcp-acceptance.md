# Chrome DevTools MCP 验收规格

## 首选工具

浏览器验收首选：

[Chrome DevTools MCP](https://github.com/ChromeDevTools/chrome-devtools-mcp)

原因：

- 控制真实 Chrome。
- 可截图。
- 可检查网络请求。
- 可检查控制台。
- 可验证 DOM、交互和视口。
- 更贴近本地 Chrome 双击打开目标场景。

Playwright 仅作为 CI 或批量截图备选。

## 必跑检查

必须检查：

- 页面可在 Chrome 打开。
- 无外部网络请求。
- 无控制台错误。
- 无整页横向溢出。
- 优化副本与源 HTML 的内容结构保持一致。
- 浅色模式截图正常。
- 深色模式截图正常。
- 一键主题切换可用。
- Tabs 可用。
- 折叠面板可用。
- 表格搜索/筛选可用。
- 表格长文本不被无提示截断。
- 数字列右对齐且等宽。

## 视口

必须覆盖：

- `1280px`
- `1366px`
- `1440px`
- `1536px`
- `1920px`

正式门禁可分两档：

- 开发阶段：至少 `1440px` 浅色 + 深色。
- 发布阶段：全部宽度浅色 + 深色。

## 验收流程

1. 打开本地 HTML。
2. 记录源文件路径和输出副本路径。
3. 记录网络请求。
4. 记录控制台消息。
5. 检查页面横向溢出。
6. 截取第一屏浅色截图。
7. 点击主题切换。
8. 截取第一屏深色截图。
9. 截取第一个表格密集区截图。
10. 截取一个长文段落区截图。
11. 操作关键交互。
12. 记录内容结构保持结论。
13. 按 `references/visual-quality-gate.md` 做截图视觉复审和评分。
14. 输出验收结论。

## 失败处理

出现以下情况必须回到实现修复：

- 控制台错误。
- 外部网络请求。
- 横向溢出。
- 深色模式不可读。
- 主题按钮无效。
- 关键交互无效。
- 表格长文本被裁剪。
- 内容结构被改写、重排、删减、合并或模板同化。
